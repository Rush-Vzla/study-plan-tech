## 1. Conceptos

### El problema de los formularios sin tipado

Sin tipado, los formularios en React se ven así:

```tsx
const handleSubmit = (e: React.FormEvent) => {
  e.preventDefault();
  const data = new FormData(e.target as HTMLFormElement);
  const amount = data.get('amount'); // string | File | null
  const parsed = parseFloat(amount as string); // casteo sin validación
};
```

Hay casteos en cada paso. Si el backend espera un número y mandas un string vacío, el error llega tarde — en el servidor, no en el cliente.

### El flujo correcto en Rush

1. Defines el schema de los datos con Zod
2. Derivas el tipo TypeScript del schema
3. Conectas React Hook Form con `zodResolver`
4. React Hook Form valida con Zod en cada cambio o al submit
5. Los errores llegan tipados — TypeScript sabe qué campos pueden tener error

```ts
const saleFormSchema = z.object({
  amount: z.number({ coerce: true }).positive('El monto debe ser mayor a 0'),
  currency: z.enum(['VES', 'USD']),
  note: z.string().max(200).optional(),
});

type SaleFormData = z.infer<typeof saleFormSchema>;
```

Fíjate en `z.number({ coerce: true })`: los inputs HTML siempre devuelven strings. Con `coerce`, Zod convierte el string al tipo correcto antes de validar. Sin coerce, tendrías que hacer la conversión manual.

### Controller para componentes no-input

shadcn/ui expone `Select`, `Checkbox`, `RadioGroup` — componentes controlados que no son un `<input>` HTML nativo. React Hook Form usa la prop `ref` para registrar los inputs, pero esos componentes no la exponen.

`Controller` resuelve eso: envuelve el componente y maneja el estado manualmente:

```tsx
<Controller
  name="currency"
  control={control}
  render={({ field }) => (
    <Select value={field.value} onValueChange={field.onChange}>
      <SelectTrigger>
        <SelectValue placeholder="Moneda" />
      </SelectTrigger>
      <SelectContent>
        <SelectItem value="VES">VES</SelectItem>
        <SelectItem value="USD">USD</SelectItem>
      </SelectContent>
    </Select>
  )}
/>
```

### Mostrar errores sin duplicar lógica

`formState.errors` es un objeto tipado que tiene los errores de cada campo:

```tsx
<Input {...register('amount')} />;
{
  errors.amount && <p className="text-sm text-red-600">{errors.amount.message}</p>;
}
```

TypeScript sabe que `errors.amount` puede ser `undefined` o un objeto con `message: string`. No necesitas castear nada.

## 2. Lab guiado

<LabSpec title="Formulario de venta con validación Zod" estimatedMinutes={60} runnable={false}>

Vas a crear el formulario para registrar una venta con validación en cliente antes de enviar al API.

### Setup

```bash
pnpm add react-hook-form@7 @hookform/resolvers@3 zod@3
```

### Paso 1: definir el schema del formulario

```ts
// src/features/sales/schemas/sale-form.schema.ts
import { z } from 'zod';

export const saleFormSchema = z.object({
  amount: z
    .number({ coerce: true, invalid_type_error: 'Ingresa un número válido' })
    .positive('El monto debe ser mayor a 0')
    .max(1_000_000, 'El monto no puede superar 1.000.000'),
  currency: z.enum(['VES', 'USD'], {
    errorMap: () => ({ message: 'Selecciona una moneda' }),
  }),
  note: z.string().max(200, 'La nota no puede superar 200 caracteres').optional(),
});

export type SaleFormData = z.infer<typeof saleFormSchema>;
```

### Paso 2: crear el formulario

```tsx
// src/features/sales/components/SaleForm.tsx
import { useForm, Controller } from 'react-hook-form';
import { zodResolver } from '@hookform/resolvers/zod';
import { saleFormSchema, type SaleFormData } from '../schemas/sale-form.schema';
import { Button } from '@/components/Button';
import {
  Select,
  SelectContent,
  SelectItem,
  SelectTrigger,
  SelectValue,
} from '@/components/ui/select';
import { Input } from '@/components/ui/input';
import { useCreateSale } from '../hooks/useCreateSale';
import { useTenant } from '@/features/auth/TenantContext';

export function SaleForm() {
  const { businessId } = useTenant();
  const createSale = useCreateSale(businessId!);

  const {
    register,
    control,
    handleSubmit,
    reset,
    formState: { errors, isSubmitting },
  } = useForm<SaleFormData>({
    resolver: zodResolver(saleFormSchema),
    defaultValues: {
      currency: 'VES',
    },
  });

  const onSubmit = async (data: SaleFormData) => {
    await createSale.mutateAsync({
      businessId: businessId!,
      amount: data.amount,
      currency: data.currency,
    });
    reset();
  };

  return (
    <form onSubmit={handleSubmit(onSubmit)} className="space-y-4">
      <div>
        <label className="text-sm font-medium">Monto</label>
        <Input type="number" step="0.01" {...register('amount')} />
        {errors.amount && <p className="mt-1 text-sm text-red-600">{errors.amount.message}</p>}
      </div>

      <div>
        <label className="text-sm font-medium">Moneda</label>
        <Controller
          name="currency"
          control={control}
          render={({ field }) => (
            <Select value={field.value} onValueChange={field.onChange}>
              <SelectTrigger>
                <SelectValue placeholder="Selecciona una moneda" />
              </SelectTrigger>
              <SelectContent>
                <SelectItem value="VES">Bolívares (VES)</SelectItem>
                <SelectItem value="USD">Dólares (USD)</SelectItem>
              </SelectContent>
            </Select>
          )}
        />
        {errors.currency && <p className="mt-1 text-sm text-red-600">{errors.currency.message}</p>}
      </div>

      <div>
        <label className="text-sm font-medium">Nota (opcional)</label>
        <Input {...register('note')} placeholder="Descripción de la venta" />
        {errors.note && <p className="mt-1 text-sm text-red-600">{errors.note.message}</p>}
      </div>

      <Button type="submit" loading={isSubmitting}>
        Registrar venta
      </Button>
    </form>
  );
}
```

### Verificación final

Prueba estos escenarios en el formulario:

1. Submit con el campo "Monto" vacío → debe mostrar "Ingresa un número válido"
2. Submit con monto negativo → debe mostrar "El monto debe ser mayor a 0"
3. Submit con datos válidos → la mutación debe ejecutarse y el formulario resetearse
