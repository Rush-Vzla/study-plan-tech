## 1. Conceptos

### Los cuatro locales de Rush

Rush soporta cuatro locales:

- `es-VE` — Español Venezuela (default)
- `es-AR` — Español Argentina
- `en` — English
- `pt-BR` — Português Brasil

Fíjate que `es-VE` y `es-AR` son el mismo idioma pero con diferencias de vocabulario y formato de moneda. Rush mantiene archivos separados para no mezclar "Bs." con "$" o "ARS".

### Namespaces por feature

i18next usa namespaces para organizar las traducciones. Rush usa un namespace por feature:

```text
public/locales/
├── es-VE/
│   ├── common.json       <- textos compartidos (botones, errores genéricos)
│   ├── sales.json        <- textos del feature sales
│   └── auth.json         <- textos del feature auth
├── es-AR/
│   └── ... (misma estructura)
├── en/
│   └── ...
└── pt-BR/
    └── ...
```

Cargar namespaces por feature en vez de uno solo global reduce el tiempo de carga inicial — el feature de ventas solo carga `sales.json` cuando se necesita.

### Formateo de fechas y números

Los números y fechas se formatean con la API nativa de JavaScript (`Intl`), no con textos hardcodeados:

```ts
const amount = 1500.5;
const formatted = new Intl.NumberFormat('es-VE', {
  style: 'currency',
  currency: 'VES',
}).format(amount);
// → "Bs. 1.500,50"

// Con locale 'en':
// → "VES 1,500.50"
```

La i18n no es solo traducir texto — es formatear según las convenciones del locale.

### Paso 2: configurar i18next

```ts
// src/lib/i18n.ts
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import HttpBackend from 'i18next-http-backend';

i18n
  .use(HttpBackend)
  .use(initReactI18next)
  .init({
    lng: 'es-VE',
    fallbackLng: 'en',
    ns: ['common', 'sales', 'auth'],
    defaultNS: 'common',
    backend: {
      loadPath: '/locales/{{lng}}/{{ns}}.json',
    },
    interpolation: {
      escapeValue: false,
    },
  });

export default i18n;
```

```tsx
// src/main.tsx — importar antes del árbol de React
import '@/lib/i18n';
```

### Paso 3: usar en componentes

```tsx
// src/features/sales/components/SalesPage.tsx
import { useTranslation } from 'react-i18next';

export function SalesPage() {
  const { t } = useTranslation('sales');

  return (
    <div>
      <h1>{t('title')}</h1>
    </div>
  );
}
```

### Paso 4: selector de idioma

```tsx
// src/components/LanguageSelector.tsx
import { useTranslation } from 'react-i18next';

const LOCALES = [
  { code: 'es-VE', label: 'Español (Venezuela)' },
  { code: 'es-AR', label: 'Español (Argentina)' },
  { code: 'en', label: 'English' },
  { code: 'pt-BR', label: 'Português (Brasil)' },
];

export function LanguageSelector() {
  const { i18n } = useTranslation();

  return (
    <select
      value={i18n.language}
      onChange={(e) => i18n.changeLanguage(e.target.value)}
    >
      {LOCALES.map((locale) => (
        <option key={locale.code} value={locale.code}>
          {locale.label}
        </option>
      ))}
    </select>
  );
}
```

### Verificación final
