## 1. Conceptos

Venezuela tiene un contexto monetario particular: los negocios operan en bolívares (VES) y dólares (USD) al mismo tiempo. Un negocio puede recibir el pago de una venta en VES, pero querer reportar sus métricas en USD.

El problema técnico: la tasa de cambio varía todos los días. Si guardas solo el monto en VES, no puedes saber cuánto valía esa venta en USD el día que ocurrió — la tasa de hoy es distinta a la de ayer.

### La 4-tupla

La solución de Rush es guardar cuatro valores por cada monto monetario:

| Campo            | Tipo          | Descripción                                        |
| ---------------- | ------------- | -------------------------------------------------- |
| `amount`         | NUMERIC(19,4) | El monto en la moneda original de la transacción   |
| `currency`       | TEXT          | La moneda del monto ('VES' o 'USD')                |
| `usd_equivalent` | NUMERIC(19,4) | El equivalente en USD al momento de la transacción |
| `reference_rate` | NUMERIC(19,6) | La tasa VES/USD usada para el cálculo              |

Fíjate que guardar `reference_rate` es lo que permite auditoría: puedes verificar exactamente con qué tasa se convirtió ese monto, sin depender de una fuente externa.

### El schema

```ts
// En sales_events o cualquier tabla con montos
amount: numeric('amount', { precision: 19, scale: 4 }).notNull(),
currency: text('currency').notNull(),
usdEquivalent: numeric('usd_equivalent', { precision: 19, scale: 4 }),
referenceRate: numeric('reference_rate', { precision: 19, scale: 6 }),
```

Cuando el pago es en USD, `usd_equivalent` == `amount` y `reference_rate` == 1.

### La tabla exchange_rates

```ts
// src/db/schema/exchange-rates.ts
import { pgTable, uuid, numeric, text, timestamp } from 'drizzle-orm/pg-core';

export const exchangeRates = pgTable('exchange_rates', {
  id: uuid('id').defaultRandom().primaryKey(),
  fromCurrency: text('from_currency').notNull(),
  toCurrency: text('to_currency').notNull(),
  rate: numeric('rate', { precision: 19, scale: 6 }).notNull(),
  source: text('source').notNull(),
  effectiveAt: timestamp('effective_at', { withTimezone: true }).notNull(),
  createdAt: timestamp('created_at', { withTimezone: true }).defaultNow().notNull(),
});
```

El campo `source` documenta de dónde viene la tasa (la API del banco que use el negocio, un webhook del ERP, o el rate ingresado manualmente por el dueño del negocio). La tasa nunca se hardcodea — siempre viene de la tabla.

### Obtener la tasa vigente

```ts
async getCurrentRate(from: string, to: string): Promise<number> {
  const [rate] = await this.drizzle.db
    .select({ rate: exchangeRates.rate })
    .from(exchangeRates)
    .where(
      and(
        eq(exchangeRates.fromCurrency, from),
        eq(exchangeRates.toCurrency, to),
      ),
    )
    .orderBy(desc(exchangeRates.effectiveAt))
    .limit(1);

  if (!rate) {
    throw new DomainError('EXCHANGE_RATE_NOT_FOUND', `No rate found for ${from}/${to}`);
  }

  return parseFloat(rate.rate);
}
```

### Calcular y guardar la 4-tupla

```ts
async recordSaleWithDualCurrency(
  businessId: string,
  amount: number,
  currency: 'VES' | 'USD',
): Promise<SaleEvent> {
  let usdEquivalent: number;
  let referenceRate: number;

  if (currency === 'USD') {
    usdEquivalent = amount;
    referenceRate = 1;
  } else {
    referenceRate = await this.exchangeRateService.getCurrentRate('VES', 'USD');
    usdEquivalent = amount / referenceRate;
  }

  return this.drizzle.db.insert(salesEvents).values({
    businessId,
    amount: String(amount),
    currency,
    usdEquivalent: String(usdEquivalent),
    referenceRate: String(referenceRate),
  }).returning().then(([event]) => event);
}
```

```bash
curl -X POST http://localhost:3000/sales/biz-123 \
  -H "Content-Type: application/json" \
  -d '{"amount": 365, "currency": "VES"}'
```

La respuesta debe incluir `usdEquivalent ≈ 10` (365 / 36.50) y `referenceRate: "36.500000"`.

```bash
curl -X POST http://localhost:3000/sales/biz-123 \
  -H "Content-Type: application/json" \

```

La respuesta debe tener `usdEquivalent: "50.0000"` y `referenceRate: "1.000000"`.
